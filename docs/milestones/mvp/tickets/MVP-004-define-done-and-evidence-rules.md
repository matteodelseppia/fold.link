# MVP-004: Define ticket completion and evidence rules

## Description

Document the common definition of done for these micro-tickets, including code review, commands run, external-console evidence, documentation updates, and prohibition on combining unfinished acceptance criteria.

## Acceptance Criteria

- The definition requires changed behavior to have an automated test.
- External Infisical, GitLab, DNS, and Railway changes require reproducible notes without secret values.
- Failed or skipped checks must keep a ticket open.
- Testing: apply the checklist to one repository ticket and one hosted-service ticket; neither needs an undocumented completion step.
